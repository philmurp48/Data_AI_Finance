## Quick Start

### For New Projects

1. **Create your project directory**
   ```bash
   mkdir my-new-project
   cd my-new-project
   ```

2. **Copy this entire `get-started` folder**
   ```bash
   cp -r ../get-started ./
   ```

3. **Follow the setup checklist**
   - Read `SETUP-CHECKLIST.md` for step-by-step initialization
   - Configure branding (Accenture vs Client)
   - Set up Cursor rules and commands
   - Initialize your technology stack

4. **Start building**
   - Use templates from `templates/`
   - Follow design standards from `design-standards/`
   - Reference actual implementations in `project-examples/`

## Philosophy

This starter pack embodies these core principles:

### 1. Professional First
- Enterprise-grade quality
- Polished, production-ready
- No placeholders or emojis in production code

### 2. Flexible Branding
- Default: Accenture branding (purple #A100FF, Greater Than symbol)
- Client Mode: Easy logo and color swap
- Consistent experience across all projects

### 3. Person-Centered Design
- User needs drive design decisions
- Accessibility is non-negotiable (WCAG AA minimum)
- Weekend language - clear, conversational, jargon-free

### 4. Best-in-Class UI/UX
- Modern, clean interfaces
- Consistent navigation patterns
- Responsive across all devices
- Professional data visualization

### 5. Rapid Development
- Reusable components and templates
- Cursor AI optimized for speed
- Clear conventions reduce decisions
- Comprehensive documentation

## Accenture Brand Voice

When building Accenture prototypes, embody these brand attributes:

- **Bold** - Celebrate progress, change, value created, and success shared
- **Optimistic** - Look forward with confidence and resolve
- **Agile** - Adapt to audiences, make complex simple, be succinct
- **Human** - Celebrate ideas, creativity, knowledge, expertise, diversity of thought and voices
- **Ingenious** - Look for the new, refresh the familiar, use new processes and tools

Use "weekend language" - avoid jargon, write conversationally, make technology accessible to all.
