## üìù **Tone Guidelines**

### **1. Professional & Trustworthy**

**Principle:** Financial data demands precision and confidence

**How to Apply:**
- Use authoritative yet approachable language
- Employ precise financial terminology
- Maintain structured, professional formatting
- Convey reliability through clear communication

**Examples:**
```
‚úÖ Good: "Transaction processing with 3-tier approval workflow"
‚ùå Avoid: "Easy transaction stuff with approvals"

‚úÖ Good: "97.8% automation rate with human oversight"  
‚ùå Avoid: "Pretty good automation most of the time"

‚úÖ Good: "Month-end close status: Day 3 of 5, On Track"
‚ùå Avoid: "Close is going okay, almost done"
```

### **2. Action-Oriented**

**Principle:** Make next actions obvious with clear CTAs

**How to Apply:**
- Lead with action verbs
- Use task-focused language
- Provide clear next steps
- Focus on outcomes and results

**Examples:**
```
‚úÖ Good: "Review 12 pending journal entries"
‚ùå Avoid: "There are some journal entries to look at"

‚úÖ Good: "Complete reconciliation by EOD"
‚ùå Avoid: "Reconciliation needs to be done"

‚úÖ Good: "Post Entry", "Approve Transaction", "Generate Report"
‚ùå Avoid: "Submit", "Click Here", "Go"
```

### **3. AI-Transparent**

**Principle:** Show what AI is doing, why, and confidence levels

**How to Apply:**
- Explain AI processes clearly
- Show confidence levels when relevant
- Make AI reasoning visible but not overwhelming  
- Use "How AI..." format for explanations

**Examples:**
```
‚úÖ Good: "How AI Sequences Close Tasks & Manages Dependencies"
‚ùå Avoid: "AI handles close tasks automatically"

‚úÖ Good: "Agent confidence: 94% (High)"
‚ùå Avoid: "AI is pretty sure about this"

‚úÖ Good: "Powered by AI ‚Ä¢ 92% auto-match rate"
‚ùå Avoid: "AI magic inside!"
```

### **4. Information-Dense Organization** 

**Principle:** Controllers need lots of data, organize it clearly

**How to Apply:**
- Use progressive disclosure (summary ‚Üí detail ‚Üí transactions)
- Employ clear visual hierarchy
- Group related information logically
- Provide scanning-friendly layouts

**Examples:**
```
‚úÖ Good: "5,200 journal entries ‚Ä¢ 350 reconciliations ‚Ä¢ 85 close tasks"
‚ùå Avoid: "Lots of financial data available"

‚úÖ Good: "Status: 75% complete, Day 3 of 5, On Track ‚úì"
‚ùå Avoid: "Things are going well"
```

## üìä **Financial Terminology Standards**

### **Core Financial Terms**
- **Record-to-Report (R2R):** Primary process description
- **Month-end close:** Not "period close" or "monthly close"

## üî§ **Writing Mechanics**

### **Capitalization**
- **Product Names:** "Controller AI Workbench" (always capitalized)
- **AI References:** "AI" (always uppercase), "Agent" (capitalized when referring to specific agent)
- **Financial Terms:** "Journal Entry", "General Ledger", "Balance Sheet" (capitalize formal terms)
- **Status Labels:** "Pending Approval", "In Progress" (capitalize status names)

### **Numbers & Formatting**
- **Financial Numbers:** Use tabular-nums font, right-align, include currency symbols
- **Large Numbers:** Use appropriate separators (1,000 or 1K format based on context)  
- **Percentages:** Always include % symbol, one decimal place for precision (92.5%)
- **Dates:** Use consistent format (October 28, 2025 or 2025-10-28 for data contexts)

### **Punctuation**
- **Lists:** Use bullet points (‚Ä¢) for feature lists
- **Status Indicators:** Use checkmarks (‚úì) for completed items
- **Emphasis:** Use **bold** for important terms, *italics* sparingly
- **Quotes:** Use double quotes for UI labels ("Post Entry")
